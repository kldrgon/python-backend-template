# 测试说明

## 测试分层

```text
单元测试 (unit_tests)
  └─ 纯内存，mock 外部依赖

集成测试 (integration_tests)
  └─ 真实 DB，其他外部能力按需替身

系统测试 (system_tests)
  └─ 真实 DB + Redis + Kafka + Temporal
  └─ 通过 HTTP 和事件链验证完整流程
```

## 边界约定

- 单元测试：不访问 DB、网络、文件系统
- 集成测试：重点验证 Repository、Query、事务行为
- 系统测试：验证接口、事件、工作流、最终状态

## 当前仓库的测试特点

- Alembic 初始化链为 `framework_outbox -> pre_domain`
- Blob 默认使用本地文件存储
- Blob 系统测试会覆盖为 `LocalStorageAdapter`
- 系统测试默认不要求额外准备 MinIO

## 常用命令

```bash
# 单元测试
pytest tests/unit_tests/

# 集成测试
pytest tests/integration_tests/

# 系统测试
pytest tests/system_tests/

# 全量测试
pytest tests/
```

## 测试隔离

- 每次测试启动前执行 `alembic upgrade head`
- 集成测试和系统测试会清理测试库数据
- Redis 等状态由对应 fixture 负责清理

## 何时补测试

- 新增领域规则时优先补单元测试
- 新增持久化行为时优先补集成测试
- 新增关键业务链路时补系统测试
