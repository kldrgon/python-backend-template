# 领域实现模板

## 目标

本文件提供一个可复制的实现骨架，用于在新上下文中快速落地领域层、应用层、适配层和读侧。

## 推荐目录

```text
app/<context>/
├── domain/
│   ├── aggregate/
│   ├── command/
│   ├── entity/
│   ├── event/
│   ├── exception/
│   ├── factory/
│   ├── repository/
│   ├── usecase/
│   └── vo/
├── application/
│   ├── dto/
│   ├── exception/
│   ├── port/
│   └── service/
├── adapter/
│   ├── input/
│   └── output/
├── query/
└── container.py
```

## 分层职责

- `domain/aggregate/`：聚合根和领域行为
- `domain/entity/`：从属实体
- `domain/vo/`：值对象
- `domain/repository/`：仓储接口
- `application/service/`：用例编排
- `adapter/output/`：ORM、外部系统、缓存、文件存储实现
- `query/`：面向读取场景的查询服务

## 实现约定

- 业务规则尽量放聚合内
- 应用服务只做编排，不承载核心业务规则
- 仓储负责持久化，不生成业务规则
- 事件由聚合或应用层在明确时机产生
- 读侧 DTO 不暴露基础设施细节

## 新增一个上下文时的最小步骤

1. 建立目录骨架
2. 定义聚合根和值对象
3. 定义仓储接口和用例接口
4. 实现应用服务
5. 实现输出适配器
6. 暴露 API 或事件处理入口
7. 补单元测试和必要的集成测试

## 何时保持轻量

如果上下文很小，可以先保留最小结构：

- `domain/aggregate/`
- `application/service/`
- `adapter/output/repository/`
- `adapter/input/api/`

等规则和复杂度上来后，再拆细 `entity/`、`factory/`、`query/` 等目录。
